# 0. Czym ma być Better BDOS

WILQ ma być jednym systemem operacyjnym marketera, a nie zbiorem raportów i promptów.

```text
ŹRÓDŁA I WIEDZA
GSC · GA4 · Ads · Merchant · WordPress · Ahrefs · Localo · wiedza Ekologus
                              ↓
DECYZJA DNIA → jedna priorytetowa praca, dowody, blocker i bezpieczny krok
                              ↓
OPERATORZY
Treści i SEO · Ads · Analityka · Produkty · Lokalnie · Social
                              ↓
BEZPIECZNE WYKONANIE
podgląd → review człowieka → potwierdzenie → audit → pomiar → propozycja nauki
```

## Jak ma wyglądać dobra sesja

Marketer mówi: „Chcę poprawić stronę usługi BDO”. WILQ powinien:

1. znaleźć stronę, popyt, duplikaty, aktualną treść i właściwy profil usługi;
2. pokazać decyzję: odświeżyć, scalić, utworzyć albo zablokować;
3. przygotować Sales Brief i listę dozwolonych/wymaganych/zabronionych twierdzeń;
4. przeprowadzić przez mapę sekcji i dokładną rewizję tekstu;
5. dać Codexowi ograniczone zadanie, a wynik oddać człowiekowi do oceny;
6. po akceptacji utworzyć wyłącznie szkic na devie;
7. po publikacji poczekać na okres pomiaru i zaproponować wniosek — nigdy samemu ogłosić sukces.

Ten sam wzorzec dotyczy Ads, Merchant, GA4, Localo i social: dowód → decyzja → bezpieczna akcja → pomiar. API WILQ jest mózgiem; dashboard i skills są dwoma interfejsami tego samego kontraktu.

## Kryterium Better BDOS

To nie liczba ekranów. Marketer ma szybciej podjąć dobrą decyzję, nie wymyślać metryk, nie przepuszczać ryzykownego claimu i nie wykonywać niejawnej zmiany u dostawcy.
